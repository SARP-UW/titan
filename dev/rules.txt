# Style Rules
- Everything is camel case
- Types have trailing _t
- All public variables, types, functions and macros should be prefixed with ti_
- Constants (macros, enums, static const) are all caps camel case.
- Keep lines shorter then 100 characters (wrap).
- No typedefs
- Seperate code into sections

# Error Handling
- Use ASSERT() to verify arguments passed to functions (preconditions) (MAY CHANGE...)
- Return an error code (ti_errc_t) from a function for runtime errors.
- Log an error every time a function returns an error code.
